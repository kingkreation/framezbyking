@channel Frontend Stage 4 Task Overview
Build a mobile social application called Framez using :iphone: React Native. The app allows users to share posts. Each user has a profile where they can view their own posts and activity.
This task focuses on building a functional, visually clean, and auth-integrated mobile app that demonstrates your ability to manage real-time data, authentication, and UI structure.
Core Objectives
Implement user authentication (sign-up, login, logout) using Firebase, Supabase, Convex, or Clerk.
Allow users to create posts.
Display a feed of posts from all users.
Display the current user’s profile with their posts
Key Features
Authentication
Secure login, registration, and logout flow.
Persistent user sessions (remain logged in after reopening the app).
Posts
Create and upload posts containing text and/or an image.
Display all posts in a chronological or most-recent-first feed.
Each post should display:
Author’s name
Timestamp
Profile
Display the logged-in user’s information (name, email, avatar if available).
Show all posts created by the current user.
Technical Requirements
Framework: React Native (Expo recommended).
Design: You can use Instagram as an inspiration
Backend: Firebase / Supabase / Convex / Clerk (choose one).
Database: Use your chosen backend’s real-time or document storage.
State Management: Context API, Zustand, or Redux (optional).
Deployment:
Expo Go for testing on Android/iOS simulators.
Host on http://appetize.io/
Deliverables
A fully functional mobile app built with React Native.
A public GitHub repository with:
Clean commit history.
Setup instructions in a README.md file.
Clear explanation of chosen backend and features implemented.
A 2–3 minute demo video showing app functionality.
Hosted http://appetize.io/ link
Acceptance Criteria
User can register, log in, and log out successfully.
Auth session persists on app restart.
User can create new posts.
Posts display correctly in a feed.
User’s profile displays correctly with their posts
Smooth navigation and responsive layout.
App runs without errors on Android and iOS.
Evaluation Metrics
Feature completeness and stability.
Proper implementation of authentication and data handling.
Code clarity and organization.
UI/UX design quality and responsiveness.
Creativity and polish in final presentation.
Quality of documentation and demo.
